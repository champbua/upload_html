Step 1 of 6
Create a GitHub Account
Free forever — no credit card needed
🗄️
What is GitHub? Think of it as a free storage locker on the internet for your files. GitHub Pages then takes your HTML file and turns it into a live website with a real URL you can share.
  • 1
    Go to github.com/signup in your browser
  • 2
    Enter your email address
  • 3
    Choose a username — this will appear in your website URL, so pick something clean (e.g. yourname)
  • 4
    Create a password
  • 5
    Verify your email and complete the setup
💡
Your username becomes part of the URL: yourusername.github.io/project/ — choose it carefully.

Step 2 of 6
Create a Repository
Your project's home on GitHub
📦
What is a repository? It's like a named folder that holds your project files. Each website you make gets its own repository.
  • 1
    Click the + icon in the top right corner of GitHub
  • 2
    Select "New repository"
  • 3
    Give it a name — use lowercase, hyphens only (e.g. my-travel-page)
  • 4
    Set visibility to Public
  • 5
    Leave all other settings as default
  • 6
    Click "Create repository"
ℹ️
Public vs Private: GitHub Pages requires a Public repository on the free plan. Anyone with the link can view your site, but it won't appear in search results unless indexed.

Step 3 of 6
Upload Your HTML File
Drag and drop — no command line needed
  • 1
    Open your repository on GitHub
  • 2
    Click the green "Add file" button
  • 3
    Select "Upload files"
  • 4
    Drag your HTML file into the upload area, or click to browse
  • 5
    At the bottom, type a short commit message like: Add my website
  • 6
    Click "Commit changes"
💡
What is "Commit"? It's just GitHub's way of saying "save". Every time you save changes, GitHub keeps a record so you can always go back.
ℹ️
Make sure all external resources in your HTML (fonts, images, scripts) use full URLs (starting with https://). Relative paths won't work on GitHub Pages unless the files are also in the same repository.

Step 4 of 6
Rename to index.html
So GitHub Pages knows which file is your homepage
🏠
Why index.html? It's a web convention — when someone visits a folder URL, the server automatically looks for a file named index.html and serves it as the homepage.
  • 1
    In your repository, click on your uploaded HTML file
  • 2
    Click the pencil icon (edit) in the top right
  • 3
    Click the filename field at the top of the editor
  • 4
    Clear the current name and type index.html
  • 5
    Scroll down and click "Commit changes"
💡
Alternatively, rename the file to index.html on your computer before uploading — saves one step entirely.

Step 5 of 6
Enable GitHub Pages
One toggle — then your site goes live
  • 1
    Go to your repository and click the "Settings" tab
  • 2
    In the left sidebar, scroll down and click "Pages"
  • 3
    Under "Build and deployment", confirm Source is set to "Deploy from a branch"
  • 4
    Set branch to "main" and folder to "/ (root)"
  • 5
    GitHub saves automatically — wait 1–2 minutes
  • 6
    Refresh the Settings → Pages page — you'll see your live URL appear
# Your live URL will look like this:
https://yourusername.github.io/repository-name/
✅
It's live! Bookmark this URL. Share it. It's accessible 24/7 from anywhere in the world — for free.

Common Questions
My site shows a 404 error ▼
Wait 2–3 minutes after enabling GitHub Pages. Make sure the file is named exactly index.html (lowercase) and is in the root of your repository.
Images or fonts aren't showing up ▼
All external resources must use full https:// URLs. Relative paths (like ./image.png) only work if those files are also uploaded to the same repository.
Can I add more pages to my site? ▼
Yes! Upload any additional HTML files to your repository. They'll be accessible at yourusername.github.io/repo-name/filename.html.
Can I use a custom domain? ▼
Yes. In Settings → Pages, you can add a custom domain (e.g. mytrip.com). You'll need to own the domain and configure your DNS settings to point to GitHub's servers.
